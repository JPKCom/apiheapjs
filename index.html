Download

apiheap.js is a JavaScript library that lets you make REST requests in under five lines of code.

Easy.

Apiheap.js is super easy to use. Start making your REST requests to a number of platforms, in just a few minutes.

Lightweight

Open Source

Setup

To use apiheap.js, include it after your </body> tag. Note that apiheap.js requires jQuery to function.

<script src="https://code.jquery.com/jquery-2.1.4.min.js"></script>
<script src="js/apiheap.min.js"></script>

If you prefer, you can also use apiheap.js through a CDN.

<script src="http://www.ucarecdn.com/e1523426-7a45-4d26-9b6e-78cb79c15cbb/apiheapmin.js"></script>

How It Works .

To make a REST request using apiheap.js, make an object for apiheap, and in the parameters, specify which of the supported platforms you'd like to make a request to, and an API Key/ID if required. Currently, apiheap.js supports Bitly, Tumblr, Imgur, Reddit, YouTube and OpenWeatherMap requests. All require an API Key or ID except Reddit, in which case the second param can be left blank. Also, optionally create a variable (myData) for future use. 

var myObj = new apiheap("imgur", "IMGUR_CLIENT_ID"), myData;
Next, use your object to call the method for the platform you are making a request to. The parameters for this method vary for each platform, and you can read more about them by reading the details for the platform below. In all cases, the parameters will require details about your request. For instance, the Imgur example below can take three parameters;tag_name, opt_params, and is_topic, which mean the tag to search in, any specifics for the request, and whether or not you are requesting a topic.
var myObj = new apiheap("imgur", "IMGUR_CLIENT_ID"), myData;
myObj.imgur("Awesome", "top/all", true); //Look in topic 'Awesome', search top of alltime, request is a topic.
After the aforementioned code has been executed, through either a function or a page load, your request has been made! Your data will be in myObj.RESPONSE. You can manipulate or use the data in any way that you chose, or, you can optionally use one of the functions included with apiheap.js, to parse it. In almost all cases, the function to parse data takes two parameters; the response, and the item you want. imgurParse however, takes three parameters; the response, the item you want, and whether or not the request was for a topic. In this example, the function imgurParse is being used to retrieve an array of links for images fetched from a request to the topic 'Awesome.' Remember to write this line of code inside some function that you can call after the previous lines of code have surely executed.

var myObj = new apiheap("imgur", "IMGUR_CLIENT_ID"), myData;
myObj.imgur("Awesome", "top/all", true); //Look in topic 'Awesome', search top of alltime, request is a topic.
function myFunction() {
    myData = imgurParse(imgurObj.RESPONSE, "link", true); //Passing in response (imgurObj.RESPONSE), the item we want (link), and if the request was a topic (true).
    console.log(myData); } //Use the data (array)

 //When myFunction() is triggered, bitlyLink will hold a short link for "https://google.ca"
var bitlyObj = new apiheap("bitly", "MY_AUTH_TOKEN"),bitlyLink;
bitlyObj.bitly("https://google.ca");
